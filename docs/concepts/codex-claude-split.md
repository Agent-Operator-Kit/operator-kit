# Codex And Claude Split

A common default split:

- Codex owns backend, contracts, data models, release, automation, and integration.
- Claude Code owns scoped UI composition, styling, and presentational changes.
- The operator owns final review and merges.

This is a default, not a hard rule. The important rule is explicit ownership: each task packet should say which lane owns which files and where shared seams require operator review.
